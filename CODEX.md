# Codex Notes For Pretty Planner

- Always choose the simplest direct edit.
- Use existing variables first.
- Do not create new variables unless explicitly asked.
- Do not use `color-mix()`.
- Opt for simpler concepts, easy for an entry-level junior developer to grasp.
- Do not convert hex colors into RGB helper variables.
- If transparency is needed, use variables, and append with transparency value.
- Do not keep archive/legacy code.
- Do not refactor unrelated code.
- Do not change nearby systems unless requested.
- For CSS, prefer existing gray variables: `--color-gray1` through `--color-gray4`.
- If we have an opportunity to use an existing variable instead of raw code, please point it out.
- Do not add or remove "Note" to the beginning of comments.
